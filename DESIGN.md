# timerge — 근무시간 계산 앱 설계 문서

> 회사 근무시간을 기록·계산하고, **남은 목표를 어떻게 배분할지 역산**해주는 앱.
> FLEX(근태앱)가 "지금까지 얼마 했나"(누적)를 보여준다면, timerge는 **"앞으로 어떻게 채울까 / 오늘 몇 시에 퇴근 가능한가"(계획)** 를 제공한다.

상태: 설계 합의 완료, 개발 착수 전.

---

## 0. 플랫폼 & 기술 방향 (확정)

| 항목 | 결정 |
|---|---|
| 코드베이스 | **React 웹앱** (단일 코드) |
| 모바일 출시 | **Capacitor**로 래핑 → iOS/Android 앱스토어 |
| 데스크톱 | 같은 React 코드를 웹으로 호스팅 (캡처 Ctrl+V 붙여넣기) |
| 저장 | **로컬 IndexedDB** (1단계는 기기 로컬, 동기화 없음) |
| 백업 | JSON 내보내기/가져오기 |
| OCR | 단계적: 웹 = Tesseract.js / 모바일 = 네이티브 ML Kit |
| 동기화 | **2단계로 분리** (계정 + 클라우드) — 1단계엔 미포함 |

주 입력 환경은 **모바일**. 데스크톱 웹은 보조. 동기화 전까지 기기 간 데이터는 분리된다.

## 1. 계산 규칙

### 1.1 핵심 원칙

- **모든 시간은 정수 "분(minute)"으로 저장**하고, 표시할 때만 `N시간 M분`으로 포맷한다. (반올림 없음 — 8.33시간 같은 소수 오차가 원천 차단됨)
- 계산 엔진이 읽는 **유일한 값은 각 날의 `recognizedMinutes`(인정 근무시간)** 다.
  - 수동 입력: 아래 day-type 규칙으로 `recognizedMinutes`를 계산해 채운다.
  - 캡처(OCR): **FLEX 배지값을 그대로 신뢰**해 `recognizedMinutes`에 넣는다.
  - 출퇴근 시각·세그먼트는 **메타데이터**(표시 + 마지막날 퇴근시각 역산용)일 뿐, 계산의 진실원천이 아니다. → "배지 신뢰"와 "시각 재계산"이 갈라지는 문제를 방지.

### 1.2 점심(휴게) 차감

- 점심 휴게는 **시간대와 무관하게 무조건 1시간(60분)**, 실제 출퇴근 근무가 있는 날 **하루 1회** 차감한다.
- **예외 — 오전 반차/오전 연차**: 오전 휴가의 자리비움(예 09:00~14:00 = 5h)에 점심이 이미 포함되므로, **그날 추가 근무에는 점심을 차감하지 않는다.**

판정 트리거(수동 입력 시): 같은 날 근무 세그먼트가 휴가 세그먼트보다
- **시간상 뒤**에 오면(= 오전 반차) → 근무에 **점심 차감 없음**
- **시간상 앞**에 오면(= 오후 반차) → 근무에 **점심 차감 있음**

### 1.3 근무 유형별 인정시간

| 유형 | `recognizedMinutes` | 비고 |
|---|---|---|
| 근무(오피스) | `(퇴근 − 출근) − 60` | 점심 1h 차감 |
| 외근 | `(퇴근 − 출근) − 60` | 계산은 근무와 동일, **기록용 라벨** |
| 연차(종일) | `480` (8h 고정) | 별도 계산 없음 |
| 반차(반일연차) | `240` (4h 고정) | 같은 날 추가 근무 시 1.2 규칙 적용 |
| 공휴일 | — (누적 **0**, `0:00` 표시) | 목표일수에서 **제외**. 실제 출근 시 클릭해 입력 가능 |
| 주말 근무 | `(퇴근 − 출근) − 60` | total에 합산 (§1.5) |

**자정 넘김**: 퇴근시각 < 출근시각이면 `퇴근 += 1440분`(24h) 후 계산. (드묾)

**정방향/역방향 공식 (서로 역관계):**
```
recognized = (out − in) − 60        # 인정시간 산출
departure  = in + needed + 60       # 필요시간(needed)을 채우는 퇴근시각
```

### 1.4 주간 목표 모델 (B 방식 — 공휴일 제외 표시)

- **기준 목표(baseGoal)**: 5일 만근 기준. 디폴트 `40h`. 연장근무 시 유저가 `52h` 등으로 변경.
- **이번 주 목표 `goal = baseGoal − (공휴일수 × 8h)`**. → 공휴일 1개면 `40 − 8 = 32h`, 연장 주는 `52 − 8 = 44h`.
- **공휴일**: 목표를 8h 줄이고 누적에는 **0** 기여(`0:00` 표시) — FLEX와 동일.
- **연차·반차**: `recognizedMinutes`(480·240)로 누적에 기여.

```
totalRecognized = Σ recognizedMinutes              # 공휴일 0
totalAccounted  = Σ_근무가능일( 실적 recognizedMinutes
                              else 고정목표 fixedTargetMinutes
                              else 0 )
remaining(남은 실근무) = goal − totalAccounted
미정일수 = 근무가능일(평일·비공휴일) 중 실적·고정목표 둘 다 없는 날 수
평균 필요시간 = remaining / 미정일수
초과근무 = max(0, totalRecognized − goal)
```

> **double-subtract 금지** (지난 합의의 모순 해결): 공휴일은 "목표 −8h + 누적 0" **한 방식으로만** 처리한다. 목표를 줄이면서 동시에 공휴일 8h를 누적에 더하면 평일 부담이 잘못 줄어든다(공휴일 1주에 6h/일이 나오는 버그).
> 검증: 공휴일 1주 → 목표 32h, 평일 32h면 남은 0 = **8h/일** ✓ / override 52h + 공휴일 1 → 목표 44h, 평일 44h ✓.

### 1.5 초과근무 · 주말

- **초과근무 = `max(0, 총 인정시간 − 목표)`**. "N시간 초과"로만 표시(수당 금액 계산 없음).
- **주말 근무**는 목표일수에 안 들어가지만, 인정시간은 **총합에 포함** → 목표를 먼저 채우고 넘치는 만큼 초과근무로 잡힌다. (별도 주말 버킷 없음)

### 1.6 역산 (앱의 핵심 가치)

각 날은 3가지 상태 중 하나:
1. **실적** — `recognizedMinutes`가 채워진 과거/오늘
2. **고정목표** — 유저가 "이 날은 N시간"으로 지정한 미래 (`fixedTargetMinutes`)
3. **미정** — 역산 대상

- **남은 날 평균**: `remaining / 미정일수` 를 화면 상단에 표시.
  - 예: 목표 40h, 월 공휴일(자동 8h), 금 5h 고정 → `(2400 − 480 − 300) / 3 = 540분` → **화·수·목 9h씩**.
- **마지막 근무일 퇴근 가능 시각**: 출근시각 입력 시 `퇴근 = 출근 + needed + 60`.
  - "마지막 근무일"은 요일 하드코딩이 아니라 **그 주 마지막 근무가능일**(공휴일/연차면 그 전날로).
  - 예: 마지막날 needed=5h, 09:30 출근 → `09:30 + 5h + 1h = 15:30` 이후 퇴근 가능.

### 1.7 검증 예시 (FLEX 5.25~5.31 주, 목표 40h)

| 날 | 입력 | recognizedMinutes |
|---|---|---|
| 25 월 | 공휴일(부처님오신날 대체) | — (0, `0:00`) |
| 26 화 | 오전 반차(09:00~14:00) + 근무(14:00~18:30) | 240 + 270 = **510** (오후근무 점심 차감 없음) |
| 27 수 | 근무 09:55~19:00 | (545 − 60) = **485** |
| 28 목 | 근무 09:56~18:21 | (505 − 60) = **445** |
| 29 금 | 외근 09:00~18:00 | (540 − 60) = **480** |

총 인정 = 510+485+445+480 = **1920분(32h)**, 공휴일은 0. 목표 = `40 − 8 = ` **32h** → 달성, 남은 0, 초과 0. (FLEX 화면의 "32시간"과 동일)

## 2. 데이터 모델 (IndexedDB)

> 라이브러리: Dexie.js 권장. 모든 레코드에 `id` + `updatedAt`을 둬서 **2단계 동기화를 마이그레이션 없이** 붙일 수 있게 한다.

```ts
// 설정 (단일 레코드)
Settings {
  defaultWeeklyGoalMinutes: number   // 기본 2400 (40h)
  dailyStandardMinutes:    number    // 480 (8h)
  lunchMinutes:            number    // 60
  coreTimeStart:           string    // "10:00" (표시/안내용, 계산 영향 없음)
  coreTimeEnd:             string    // "16:00"
  weekStartsOn:            "monday"
}

Week {
  id:          string      // 예 "2026-W22"
  startDate:   string      // "2026-05-25" (월)
  endDate:     string      // "2026-05-31" (일)
  goalMinutes: number      // 이 주 목표 (디폴트=설정값, override 가능)
  updatedAt:   number
}

Day {
  id:                 string          // uuid
  weekId:             string
  date:               string          // "2026-05-26"
  recognizedMinutes:  number | null   // ← 계산의 단일 진실원천 (실적)
  fixedTargetMinutes: number | null   // 미래 고정목표 (계획)
  isHoliday:          boolean
  holidayName:        string | null
  segments: Array<{                   // 메타데이터 (표시 + 역산 입력)
    type:        "work" | "field" | "annual" | "halfday"
    startMin:    number | null        // 자정 기준 분 (예 9:55 → 595)
    endMin:      number | null
    breakMinutes:number               // 보통 60, 오전반차 흡수 시 0
  }>
  source:    "manual" | "ocr"
  updatedAt: number
}

// 공휴일: 정적 base + 유저 override 2계층 (JSON 갱신해도 토글 안 날아감)
HolidayBase     { date: string; name: string }              // 앱 내장(읽기전용)
HolidayOverride { date: string; isHoliday: boolean; name?: string; updatedAt: number }
```

`recognizedMinutes`가 채워져 있으면 그 값을 쓰고, `segments`는 표시·역산용 보조. 복합일(반차+근무)은 `segments`에 각 구간을, `recognizedMinutes`에 합계를 둔다.

## 3. 화면 흐름

```
[주간 현황] ──선택──> [일 입력/편집] ──> (저장) ──> [주간 현황]
     │                      ├─ 수동 입력: 유형·출퇴근/인정시간·휴게
     │                      └─ 캡처 입력: 이미지 붙여넣기/사진선택 → OCR
     │                                     → 파싱결과 표시 → 수정 → 저장
     ├──> [설정] 주 목표 / 공휴일 토글 / 백업(내보내기·가져오기)
     └──> 이전/다음 주 이동
```

**주간 현황 (메인)** — 화면 상단에:
- 헤드라인: `누적 X / 목표 Y · 남은 Z`
- **남은 날 평균 필요시간** (예 "화·수·목 9h씩")
- **마지막 근무일 퇴근 가능 시각** (출근 입력 시)
- 초과근무 시 "N시간 초과" 뱃지
- 아래: 요일별 카드 리스트 (유형 아이콘 / 인정시간 / 출퇴근)

**캡처 입력**: 데스크톱=Ctrl+V 붙여넣기, 모바일=사진첩/카메라 선택. OCR로 추출 → **항상 수정 가능**한 폼으로 표시 후 저장. 추출 대상: 주 범위 / 일별(날짜·유형·인정시간·출퇴근·공휴일명). 상단 누적·목표 숫자는 OCR 안 함(우리가 일별값으로 재계산).

## 5. 확정된 규칙 요약 (빠른 참조)

- 점심 60분 무조건 차감, 하루 1회. 오전 반차 있는 날은 추가 근무에 차감 안 함.
- 외근 = 근무와 계산 동일(기록용 라벨). 연차=480 고정. 반차=240. 공휴일=480 자동(수정 가능).
- 주 목표 `goal = baseGoal(디폴트 40h) − 공휴일수×8h`. 공휴일=목표 −8h + 누적 0(`0:00`). 연차·반차는 누적 기여. `남은 = goal − 누적/고정`. **double-subtract 금지.**
- 초과 = max(0, 누적 − 목표). 주말 근무는 총합에 포함.
- 분 정수 저장, 표시만 포맷. 자정 넘김은 +1440.
- 마지막 근무일 = 그 주 마지막 근무가능일(요일 하드코딩 X).

1. ~~공휴일 주 헤드라인 표시~~ → **(B) 확정**: 공휴일은 목표에서 −8h, 누적 `0:00`. FLEX와 동일하게 "누적 32h / 목표 32h"로 표시.
2. 동기화 시점·방식 (계정 체계, 백엔드 선택) — 2단계 진입 시.
3. OCR 정확도 확인 후 클라우드 OCR 전환 여부.
4. 앱 이름/브랜딩 (현재 코드명 `timerge`).
```
