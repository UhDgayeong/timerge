## 🤖 Claude Code 협업 가이드

이 프로젝트는 Claude Code(AI 코딩 어시스턴트)를 사용해 개발합니다.
프로젝트 루트의 `CLAUDE.md`에 Claude Code 동작 지침이 정의되어 있습니다.

### 💬 자주 쓰는 프롬프트

| 목적 | 예시 프롬프트 |
|---|---|
| 할 일 확인 | `해야 할 일 알려줘` |
| 결정 히스토리 조회 | `왜 공휴일 데이터를 이런 식으로 구현했어?` |
| 계산 로직 질문 | `점심 차감 규칙이 정확히 어떻게 돼?` |
| 작업 종료 | `이 작업 종료해` |

> 세션을 시작하면 Claude Code가 `memory/` 파일들 및 GitHub 이슈를 자동으로 읽어 프로젝트 컨텍스트를 파악합니다.
> 별도로 설명하지 않아도 현황, 규칙, 히스토리를 바탕으로 작업합니다.

### ✅ 작업 종료 프로토콜

**하나의 작업 단위가 끝나면 반드시 종료 프로토콜을 실행하고 새 세션을 여세요.**
토큰 절감 및 컨텍스트 관리를 위해 작업 단위마다 세션을 분리하는 것을 권장합니다.

종료 시 Claude Code에게 ①**'작업'** 과 ②**'종료'** 또는 **'마무리'** 이 두 가지 키워드를 함께 말하면 작업 종료 프로토콜이 실행됩니다.

예시)
```
이 작업 종료해
작업 마무리해줘
```

자동 실행 내용:
1. `memory/implementation_status.md` 업데이트 — 완료 항목 반영
2. 다음 작업 우선순위 재정렬
3. 비자명한 결정이 있으면 `memory/decisions.md`에 항목 추가
4. `memory/MEMORY.md` 인덱스 최신화
5. 미커밋 변경사항 커밋 후 `git push`
6. 해결된 이슈 또는 불필요해진 이슈 클로즈 처리

> `memory/` 폴더는 Claude Code가 세션 간 컨텍스트를 유지하기 위한 파일들로, 레포에 포함되어 공동 작업자 모두가 공유합니다.
> 직접 편집하지 않아도 되며, 작업 종료 프로토콜 실행 시 Claude Code가 자동으로 관리합니다.

### 🔧 트러블슈팅

**공휴일이 표시되지 않는 경우** — `.env.local` 파일에 API 키가 설정되어 있는지 확인하세요.
설정 방법은 [개발 시작](#-개발-시작) 섹션 참고.

### 🐛 이슈 생성 가이드

GitHub 이슈는 Claude Code를 통해 생성하는 것을 권장합니다.

**이유**: Claude Code가 실제 코드 상태를 직접 확인한 뒤 이슈를 작성하기 때문에,
재현 조건·관련 파일·맥락이 정확하고 깔끔하게 정리됩니다.

**방법**: Claude Code에게 발견한 버그나 개선 사항을 설명하면,
코드를 점검한 후 `gh` CLI를 통해 GitHub 이슈를 직접 생성해 줍니다.

**사전 설정 — GitHub 계정 연결**:
Claude Code가 이슈를 생성하려면 `gh` CLI가 GitHub 계정과 연결되어 있어야 합니다.
최초 1회 아래 명령어를 실행해 인증하세요:

```bash
gh auth login
```

브라우저가 열리면서 GitHub 계정 인증을 완료하면 됩니다.
연결 여부는 `gh auth status` 로 확인할 수 있습니다.

## ✨ 주요 기능

- 📅 요일별 근무시간 기록 (일반·외근·연차·반차)
- 📊 주간 목표 대비 누적 현황 (공휴일 자동 반영)
- 🔁 남은 날 균등 배분 역산 — "오늘부터 매일 X시간"
- 🕐 마지막 근무일 최소 퇴근 시각 계산
- 📸 FLEX 앱 스크린샷 OCR 자동 입력 (예정)

## 🛠️ 스택

- React + Vite + TypeScript
- Capacitor (iOS / Android 앱 스토어 출시)
- Dexie.js (IndexedDB 로컬 저장)

## 🚀 개발 시작

```bash
npm install
```

`.env.local` 파일을 생성하고 API 키를 입력합니다 (`.env.example` 참고):

```bash
VITE_GOOGLE_CALENDAR_API_KEY=your_api_key_here
```

API 키는 팀 내에서 공유받거나 Google Cloud Console에서 직접 발급하세요.

```bash
npm run dev      # 개발 서버
npm test         # 테스트
npm run typecheck
```
